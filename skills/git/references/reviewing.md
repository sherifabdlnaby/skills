# Review Someone Else's PR

Covers the procedure for reviewing a PR the user points you at ("Review `<PR name or link>`").

Use `gh` first, GitHub MCP only as a fallback. Apply SKILL.md voice rules to the summary and any comment you draft, and append the AI posts footer (template in `references/pull-requests.md`) to anything posted on the user's behalf, for transparency.

## Procedure

1. Read the PR description and any existing comments or code review.
2. Read context the user gave in this conversation.
3. Read relevant previous PRs or links surfaced in the description or comments.
4. Check out the PR locally if possible.
5. Understand the code.
6. Give the user a summary.
