# Publish, sign, attest

Artifact-agnostic rules for shipping what a release built. Per-artifact mechanics live in
[`artifacts/docker.md`](artifacts/docker.md) (image digest), [`artifacts/go.md`](artifacts/go.md)
(binary checksums), and [`artifacts/packaged.md`](artifacts/packaged.md) (bundle checksums).

## Rules and Best Practices:

1. **Keyless, via OIDC when possible.** Sign with `cosign` using the workflow's OIDC identity, no private key to hold
   or rotate.
2. **Attest provenance always; SBOM where it applies.** `actions/attest-build-provenance` (SLSA) over
   the published subject (an image by `subject-name`+`subject-digest`, or a checksums file) for every
   artifact. `actions/attest-sbom` for artifacts that carry dependencies — images and compiled binaries;
   a packaged bundle's checksums get provenance alone.
3. **A signature on top for registries.** A keyless cosign signature over the attestation for what lives
   in a registry; a Release asset can skip the detached signature (attestation already binds it to the
   pipeline).
4. **Sign digests, not tags.** Sign and attest the digest the build emitted. why: tags move, digests don't.

## The publish gate

Good Releases are immutable: every artifact is uploaded and signed **before** the release goes public. flip
a draft public only after. why: a half-published release must never look final. The gate holds strictly
only in the auto-tag model ([`releases.md`](releases.md#alternative-the-release-drafter-draft-model));
GitHub's trigger variants in [`artifacts/docker.md`](artifacts/docker.md#triggering-the-publish).

## The Verify + Install footer

Generated notes say *what changed*; the release job appends a fixed footer saying how to *trust and use*
the artifact. why: an unverifiable signature protects no one, and a consumer who can't find the
installation step won't run it.

- **Verify** — the exact `gh attestation verify` (provenance), `sha256sum --check checksums.txt`, and,
  for a registry artifact, `cosign verify` (OIDC issuer + identity regexp) commands, filled with the
  repo's owner/name. Generated by the workflow.
- **Install** — how a consumer installs/runs *this* artifact, per runtime (open the `.alfredworkflow`
  bundle, `docker pull` the digest, `go install` / download the binary). Authored once per repo in
  `.github/release-footer.md`.
- **Mechanic** — the release workflow's footer step
  ([`release.yml`](../assets/.github/workflows/release.yml); GoReleaser: `release.footer`). Same text
  belongs in the README so it's discoverable off a release page too.
